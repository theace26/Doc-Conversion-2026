Pipeline File Explorer — Approach

How should clicking a pipeline stat badge show you the files?

A

Inline Expandable Panel

Click a badge → file list slides open below the stats bar, right on the status page. Toggle buttons to combine categories. Never leaves the page.

Pros

  • No page navigation — fast, contextual
  • Can glance at files without losing your place
  • Toggle multiple categories in-place

Cons

  • Status page gets taller/busier
  • Limited space for complex filtering or large file lists
  • Pagination in an inline panel can feel cramped
B

Dedicated File Explorer Page

Click a badge → navigates to pipeline-files.html?status=failed. Full page with table, search, pagination, and toggle buttons for all categories.

Pros

  • Full screen real estate for file browsing
  • Room for search, sort, pagination, bulk actions
  • Clean separation — status page stays simple

Cons

  • Extra page navigation (back button to return)
  • One more HTML page to maintain
C

Hybrid — Link to Existing Pages

Each badge links to the page that already shows those files: job-detail for pending/failed, unrecognized.html for unrecognized, etc. Filters pre-applied via URL params.

Pros

  • No new pages — reuses what's already built
  • Least effort to implement

Cons

  • Fragmented — different pages for different categories
  • No unified view to toggle between categories
  • Some categories (analysis, indexed) have no existing page
