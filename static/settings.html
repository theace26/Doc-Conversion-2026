Billing & Costs

Configure how MarkFlow reports your monthly LLM spend on the Admin page's Provider Spend card. Match these settings to your actual provider invoice cycle so the running total + month-end projection reflect what you'll owe.

Day of the month your provider's billing cycle starts. Example: if your Anthropic bill closes on the 15th, set this to 15. The Provider Spend card will then sum costs from the 16th of last month through today. Capped at 28 to avoid February edge cases.

Where do rates live? The per-1M-token cost for each model lives in core/data/llm_costs.json inside the container. To update: edit the file (it's host-mounted) and POST to /api/admin/llm-costs/reload (admin only) — no container restart needed. View the current loaded table at /api/admin/llm-costs.

Setup Instructions

Claude Code (recommended):

claude mcp add markflow --transport sse http://localhost:8001/sse

Claude Desktop — add to claude_desktop_config.json:

{
  "mcpServers": {
    "markflow": {
      "command": "npx",
      "args": ["-y", "mcp-remote", "http://localhost:8001/sse"]
    }
  }
}

Claude.ai (web) — requires tunnel (ngrok/Cloudflare):

  1. Run: ngrok http 8001
  2. Go to Claude.ai → Settings → Integrations → Add Integration
  3. Paste the ngrok URL + /sse
