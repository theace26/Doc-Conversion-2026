"Pipeline" rename + the operator dashboard

First: what to call it. Then: what's on it. Mocked with the working title Activity.

1. What do we call it? Five candidates, my recommendation in purple.
Activity Recommended
Natural English. "Let me check Activity" reads exactly like "let me check what's been happening." Same word Apple uses for similar dashboards (Activity Monitor, Activity ring), GitHub uses (Activity feed), Slack uses (Activity tab).
Why: what's running, what's queued, what just finished, what failed — all of it is "activity." Doesn't lock you into any one verb.
Status
Direct. "Is it working?" If your daily question is "is anything broken right now," Status answers it head-on. Used by countless ops tools.
Concern: implies passive monitoring only. The page also has actions (pause, run-now). "Status" undersells those.
Conversions
Maps directly to the work. Plural noun. Clear signal: "this is where the conversion engine lives."
Concern: reads as a list page (like "Customers" or "Invoices"), not a dashboard. Misses the live/operational feel.
Engine
Mechanical, specific. "The conversion engine." Like Stripe Sigma's "Engine" or Shopify's "App engine." Power-user-friendly.
Concern: a bit jargon-y for IBEW staff. Better for admin-only tools, weaker for shared use.
Inbox
Borrows the email metaphor. "What came in to MarkFlow." Familiar, friendly, immediately understood.
Concern: overloads the meaning — people think "messages." Doesn't carry the running-jobs / failed-jobs side.

Mockup below uses Activity. Tell me which one feels right and I'll swap.
