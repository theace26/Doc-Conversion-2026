How would you like your home to feel?

Three options, all powered by the same Search bar. Pick the one that fits how you'll actually use MarkFlow day-to-day — you can swap anytime from the layout icon in the nav (or press ⌘\).

Maximal

Browse everything

Search bar plus shelves of cards: pinned folders, what's auto-ingested, what's most-accessed, what's flagged for review, browse by topic.

Best for
□"Friday catch-up" — surveying everything that came in this week
□"What did we already have?" — discovering pinned folders and topics
□New to MarkFlow — surfaces show you what's possible
Recent

Pick up where you left off

Search bar plus your recent searches and recently opened files. Nothing else. Quiet, fast, and always relevant to whatever you were just doing.

Best for
□"Tuesday at 10am" — finishing the same project you started yesterday
□Daily working tools — less visual noise, more flow
□Power users who want quick re-access without scanning
Minimal

Just the search box

Search bar centered, nothing else. You know what you're looking for — type and go. Calm by default.

Best for
□"Mid-meeting lookup" — search and dismiss
□Screen-sharing — nothing distracting on display
□Focused use — no visual cost when you're between tasks

4. Where this lives in the spec

Locked decisions

  • No imposed system default. First-run flow forces an active choice with concrete examples (Maximal / Recent / Minimal).
  • Minimal is the visual recommendation in the picker (purple ring) and the skip fallback. Calmest starting point — users opt up to Recent or Maximal once they know what those add.
  • Skip option ("I'll pick later") → user lands on Minimal. Easy to discover the layout icon in the nav and explore the other modes from there.
  • Nav quick-toggle — small layout icon next to avatar. Single-click cycles. Click-and-hold (or right-click) opens the chooser popover.
  • Keyboard cycle — ⌘\ on every page. Same key, anywhere.
  • "Track recent searches" — ON by default (your call). Settings → Display has the off switch.
  • Persistence — per-user preference stored in DB (or localStorage for unauthenticated/SSO bootstrap), restored on every load.
