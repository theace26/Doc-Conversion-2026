Just locked

Settings persistence + role binding

User-portable. All settings (display preferences, pinned folders, layout mode, density, snippet length, advanced visibility, recent searches) follow the user from terminal to terminal — stored against user identity, not machine. Sign in on the workstation in the union hall and your home looks the same as on your laptop.

UnionCore-bound roles. Role hierarchy (member / operator / admin) is defined upstream in UnionCore. MarkFlow consumes it via JWT. Role-gated visibility (Activity nav link, System section, API keys, advanced power-user features) all check the role claim from UnionCore.

Implication: the "All settings →" overview page is preserved as the deep-dive entry. Card grid lives on. Avatar is the daily-use shortcut into common sections.

Search = home

Home page

  • Search is the landing surface for everyone.
  • Three home modes: Maximal (browse rows), Recent (search + history), Minimal (search only).
  • No system default — first-run picker forces choice with K-drive framing.
  • Minimal = recommended + skip fallback.
  • ⌘\ cycles modes; layout-icon in nav opens chooser popover.
Cards

Document cards

  • Gradient top-band (32%) + paper snippet body. No metadata below the card.
  • Format-coded gradients (PDF red, DOCX blue, etc.).
  • Hover → rich preview popover (full meta, AI summary, action buttons).
  • Right-click → context menu (View / Export / AI / Pin-Flag) + collapsed Advanced section for Markdown actions.
  • Three densities: Cards (6/row) · Compact (8/row) · List (linear). Per-density items-per-page (12-96 / 24-192 / 100-All).
Power-user gate

Markdown / AI actions

  • Hidden behind Advanced ▾ expander in context menus by default.
  • Settings → Display → "Show MarkFlow / AI integration actions" toggle inverts the default.
  • Off by default for member, on by default for admin/operator.
  • Same toggle gates future power features (raw API access, share-links, content hash).
Folder browse

"Go to containing folder"

  • Breadcrumb · folder header with stats · density toggle · Pin / Download-all.
  • Hover-checkbox selection on cards.
  • Bulk action bar appears when items selected: Download / Preview / Copy paths / Tag / Flag / Clear.
  • Right-click menu still works on individual rows.
Avatar menu

Settings entry

  • Header: name + role pill (member / operator / admin) + scope ("IBEW Local 46").
  • Personal (everyone): Profile, Display, Pinned folders, Notifications. API keys admin/operator only.
  • System (admin only, gated badge): Storage · Pipeline & lifecycle · AI providers · Account & auth · Database health · Log management.
  • "All settings →" link — jumps to card-grid overview for deep dives.
  • Help / Shortcuts / Report bug / Sign out / build info at bottom.
Activity (admin only)

Operator dashboard

  • Status pulse + headline + tiles row (Processed today / In queue / Active jobs / Last error).
  • 24h throughput sparkline.
  • Running now — active bulk jobs with progress, pause/cancel/open, ETA.
  • Queues — Recently converted · Needs OCR · Awaiting AI summary · Recently failed.
  • Recent jobs — last 7 days, drill-down.
  • Pipeline controls — toggle, run-scan-now, logs, db health.
