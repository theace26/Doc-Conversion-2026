Cost rates: CSV format

How to import provider rates into MarkFlow. Same format works whether you maintain one CSV per provider or one combined file.

Required columns

  • provider — lowercase: anthropic, openai, local
  • model — the model ID exactly as the provider returns it
  • input_per_1m — USD per 1M input tokens
  • output_per_1m — USD per 1M output tokens

Optional columns

  • cache_write_per_1m / cache_read_per_1m — for prompt-cache providers (Anthropic)
  • vision_per_image — per-image vision rate, where applicable
  • batch_discount_pct — e.g., 50 for Anthropic batch API
  • effective_date — ISO date, in case you keep historical rates

Example

provider,model,input_per_1m,output_per_1m,cache_write_per_1m,cache_read_per_1m,vision_per_image anthropic,claude-opus-4-7,15.00,75.00,18.75,1.50,0.024 anthropic,claude-sonnet-4-6,3.00,15.00,3.75,0.30,0.0048 anthropic,claude-haiku-4-5,1.00,5.00,1.25,0.10,0.0016 openai,gpt-4o,2.50,10.00,1.25,1.25,

Notes

  • Empty cells mean "doesn't apply" — not zero. MarkFlow skips that rate type for that model.
  • You can keep multiple effective_date rows for the same model. MarkFlow uses the latest dated row not in the future.
  • CSV must be UTF-8. Excel "Save As CSV (UTF-8)" works; "CSV (Comma delimited)" usually does too.
  • Check the provider's official pricing page when you import — rates change without notice.
